# Fuzz regression corpus

Committed inputs for the `canvas_*` API harness (`../fuzz_api.c`). Unlike the
gitignored `../seeds/` scratch (regenerated by `../build.sh` for live fuzzing),
**these files are tracked on purpose**: `ninja` replays every one of them through
the harness under the `debug-cpu` sanitizers (ASan + UBSan, over the
`-fbounds-safety` software-backend core) as part of the default build
(`ninja fuzzcorpus` runs just this). A clean replay is silent; any ASan/UBSan
report or bounds trap fails the build. That makes each input a permanent,
libFuzzer-free regression — no Homebrew clang or `sudo` needed.

## What's here

- `seed00.bin`–`seed05.bin` — real drawing programs from `../seed_gen.c` (filled
  triangle, dashed stroked Bézier, arc/ellipse/round-rect fills, gradient + clip
  + transform, image-data round-trip + `drawImage`, composite modes). They give
  the replay broad renderer coverage.
- `all_ff.bin`, `all_zero.bin`, `byte_ramp.bin`, `empty.bin` — adversarial byte
  patterns. Because the decoder is total (any byte string is a valid program and
  coordinates are raw 4-byte floats), `all_ff` drives NaN/Inf/huge values into the
  path math and the `(int)` casts — the float-cast UB class behind security
  findings 4 & 6 — while `empty` exercises the zero-length edge.

## Adding a regression

When the live fuzzer (`../build.sh`) finds a crasher, minimize it and drop the
input here:

```sh
./build/fuzz/fuzz_api -minimize_crash=1 -runs=100000 <crash-file>   # reduce
cp <reduced-file> fuzz/corpus/regress-<short-description>.bin
python3 configure.py                                                # pick it up
ninja fuzzcorpus                                                    # must pass once fixed
```

Any filename ending in `.bin` is picked up automatically. Keep inputs small and
named for the bug they pin down.
